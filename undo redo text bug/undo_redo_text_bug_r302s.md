## Problem description

After removing the text and manipulating `Redo` and `Undo`, it turns out that I have text with an effect that I did not impose.

## Attach a .c3p

[undo_redo_text_bug_r302s.zip](https://github.com/WilsonPercival/WilsonPercival/files/9109298/undo_redo_text_bug_r302s.zip)

## Steps to reproduce

1. Create a new project.
2. Go to the event editor.
3. Add a text object.
4. Remove it.
5. Add an effect to it.
6. Go to the layout editor.
7. Press the `Undo` button 3 times.
8. Click the `Redo` button.

## Observed result

![bug](https://user-images.githubusercontent.com/91274932/178920570-b486be59-ef30-4bd9-8f9f-39657224c645.gif)

## Expected result

After pressing `Redo` the text has no effect.

## More details

This works with text, but doesn't work with sprites. If you delete the sprite, the properties window will immediately disappear.

**Affected browsers/platforms:** Chrome

**First affected release:** broke in r302s
