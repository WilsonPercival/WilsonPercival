## Problem description

It's impossible to assign NaN to a sprite's property X through NaN, nor through 0/0.

## Attach a .c3p

[set_nan_bug_r268s.zip](https://github.com/WilsonPercival/WilsonPercival/files/7692212/set_nan_bug_r268s.zip)

## Steps to reproduce

1. Open the project.
2. Run "Preview project".
3. Open the console.

## Observed result

A property X of the first sprite changed to NaN. A property X of the second sprite didn't.

![dfgdfg](https://user-images.githubusercontent.com/91274932/145563434-34ced264-fce2-4c9c-bcf5-b4ba9e99a518.png)

## Expected result

It should work the same, whether in events or in JS.

## More details

I can't assign NaN in JS, but I can easily do so in events. I can assign NaN to scrollX both in JS and events.

**Affected browsers/platforms:** Chrome

**First affected release:** broke in r268s, r276s
